A6 — Organizational Closure. The organism's governance governs itself.
     The rules that constrain growth are subject to the same constraints.

A8 — Topological Plasticity. The organism's structural organization is a

### Step 2: Expand Procedure 1 (Section II)

Add to Procedure 1:

**Step 1a — Classify the ontological type.** What kind of thing are you creating? Seven irreducible categories (from SPEC-002):

| Type | What it is |
|------|-----------|
| Entity | persistent object with identity |
| Value | property of an entity |
| Relation | typed connection between entities |
| Event | occurrence that changes state |
| State | configuration of entities at a point in time |
| Constraint | rule limiting valid states |
| Capability | authorized transformation |

**Step 5 expansion** — When connecting to the signal graph:
- 5a: Classify edge type (dependency / information / governance / evolution)
- 5b: Validate signal (5 conditions: class declared, producer identifiable, receiver class lawful, schema legible, interpretation burden defined)
- 5c: Check prohibited couplings (6 prohibitions — see Step 3 below)
- 5d: Declare direction (feedback = downstream→upstream; feedfront = upstream→downstream)
- 5e: If output is reusable, route to at least one preservation function

**Step 6 — Assert traceability.** The function must trace upward (to constitutional purpose, to the axiom or need it serves) and downward (to actual artifacts and outputs it produces).

**Procedure 4 (NEW) — When something moves.** Migration follows 5 modes:
- EXTRACTIVE: only a specific output is absorbed
- METHODIC: the function's method becomes standard
- STRUCTURAL: the organism gains a new internal law
- PEDAGOGICAL: the output enters teaching/curriculum
- ARCHIVAL: the function becomes preserved reference without live authority

Migration conditions: repeatable, traceable, cross-context useful, non-accidental, expressible in canonical terms.

Source: Formation Protocol §8.2, §8.4, §10, §14, §19

### Step 3: Add Section IIIa — Structural Laws (NEW)

Insert between Section III and Section IV. Six laws:

**Law 1 — Prohibited Couplings.** Six patterns that violate the axioms:

| Prohibition | Violated axiom |
|-------------|---------------|
| Unbounded mutual recursion between functions | A3 (persistence) |
| Silent ontology mutation through downstream tooling | A6 (closure) |
| Preservation layers acting as hidden governance authority | A6 (closure) |
| Routing functions inventing theory | A1 (purpose) + A5 (minimality) |
| Experimental functions bypassing review to become canonical | A6 (closure) |
| Presentation layers redefining the organism's foundation | A6 (closure) |

**Law 2 — Feedback vs. Feedfront.** Feedback = downstream to upstream (error, constraint, evidence, failure). Feedfront = upstream to downstream (primitives, schemas, aesthetic seeds). Different attenuation policies apply.

**Law 3 — Signal Preservation.** All outputs intended for reuse must route to at least one preservation function. Unrouted outputs are ephemeral.

**Law 4 — Multiplex Flow.** The organism maintains at least four independent relationship types: dependency (what must exist before what), information (what signals flow where), governance (what constrains what), evolution (what changes what). These must not collapse into one graph.

**Law 5 — Structural Interrogation.** Any function, mechanism, or decision can be examined through seven dimensions: Existence, Identity, Structure, Law, Process, Relation, Teleology. If a subject cannot answer all seven clearly, it is structurally incomplete.

**Law 6 — Meta-Evolution.** Four strata of change: State (variables change), Structure (topology changes), Ontology (categories change), Meta-Evolution (the rules of change change). Higher strata require greater governance rigor.

Source: Formation Protocol §10.3, §10.1-2, §8.4; Multiplex Graph Model; Structural Interrogation; Meta-Evolution Architecture

### Step 4: Expand Section III — Structural Properties

Add four subsections derived from the new axioms:

- **From A6 (closure):** The health checks check themselves. The naming convention names itself. The signal graph graphs itself. Any governance mechanism that exempts itself from governance is structurally invalid.
- **From A7 (primacy):** No structural optimization may override the purpose of serving the individual. System efficiency that reduces operator capability is a structural defect.
- **From A8 (plasticity):** Functions may reorganize, mechanisms may fuse/split/dissolve — all through governed evolution. The organism's topology is never frozen.
- **From A9 (inheritance):** When a function is removed, renamed, merged, or split, its identity chain must be preserved. The organism's memory of what it was is part of what it is.

### Step 5: Expand Section IV — Health Checks (11 → 15)

Add four checks:

```
CHECK 12 — SIGNAL VALIDITY
  ∀ signals s in active connections:
    class(s) declared ∧ producer(s) identifiable ∧ receiver(s) lawful ∧
    schema(s) legible ∧ interpretation(s) defined
  Violation → invalid signal in the graph. Declare missing properties.

CHECK 13 — PROHIBITED COUPLINGS
  No prohibited coupling pattern (Law 1) exists in the signal graph.
  Violation → structural prohibition violated. Break the coupling.

CHECK 14 — MULTIPLEX SEPARATION
  Dependency, information, governance, and evolution edges are tracked
  as independent relationship types, not collapsed.
  Violation → edge types conflated. Separate them.

CHECK 15 — TRACEABILITY
  ∀ functions f: upward_trace(f) reaches a constitutional purpose ∧
  downward_trace(f) reaches at least one artifact or output.
  Violation → ungrounded function. Establish trace.
```

### Step 6: Expand Section V — Convergence (3 → 10 predictions)

Add seven convergence predictions:

**Convergence 4: Formation differentiation.** Mature organisms converge on at least 7 function archetypes: generators (produce primitives), transformers (convert material), routers (coordinate flow), reservoirs (preserve/curate), interfaces (mediate boundaries), laboratories (bounded experimentation), synthesizers (combine across domains).

**Convergence 5: Graph decomposition.** The single signal graph decomposes into at least 4 independent edge families. Composite diagnostic indices emerge for each.

**Convergence 6: Ontological self-model.** The mature organism develops a multi-layer self-index covering identity, hierarchy, relation, state, event, and governance. This index is the organism's model of itself. It senses structural drift, infers implications, and evolves its own structure.

**Convergence 7: Developmental phases.** Growth proceeds through regimes: emergence → design → embodiment → stabilization → completion/evolution. Each regime has diagnostic questions and transition predicates. The organism discovers its phase by checking predicates, not by following a schedule.

**Convergence 8: Formation lifecycle.** Functions develop a lifecycle: proposed → incubating → active → under review → migrating/buffered → archived/dissolved. A young organism has only active functions. The lifecycle emerges with governance capacity.

**Convergence 9: Self-rendering.** The mature organism renders its own state into perceptible form. Every function becomes simultaneously a generative function — producing artifacts that make internal state visible.

**Convergence 10: Temporal cadence differentiation.** Multiple temporal modes emerge: event-driven, periodic, continuous, archive-triggered, review-gated, hybrid. A young organism operates in one mode.
