### River 2: STYX (Commit)
**What it means for V2:** Once a formation enters the distillation pipeline, it's committed to V2 governance. No halfway states. The formation either passes all four stages and enters the new org, or it doesn't enter at all.

**Implementation:** Stage 3 (citrinitas) — the formation is tested against V2 governance:

**Implementation:** V1 orgs are archived. DNS/redirects point to the new org. The registry is rebuilt from V2 formations only. The old registry-v2.json becomes historical record, not governance.

## The Infection Prevention Protocol

The specific danger: V1 patterns replicating into V2 through context windows.

**Prevention mechanisms:**

1. **Clean-room seed generation.** V2 seed.yaml files are NEVER generated by reading V1 seed.yaml files. They are generated from: (a) reading the actual source code, (b) computing signal signatures from actual imports/exports, (c) applying the formation chemistry bonding rules. The V1 seed is not an input.

2. **Constitutional checkpoint.** Every V2 formation must pass a constitutional compliance check BEFORE entering the new org: AX-1 through AX-8, LEX-I through LEX-X, LIM-I through LIM-III, PROHIB-I. If it violates any law, it doesn't enter.

3. **Polis review.** The constructed polis (AX-8) reviews the formation BEFORE it enters V2. The polis includes a "V1 contamination auditor" lens that specifically checks: does this formation carry V1 assumptions? Are there hardcoded organ numbers instead of named functions? Are there V1 registry references? V1 naming conventions?

4. **Batch isolation.** Formations are distilled in small batches (5-10), not all 127 at once. Each batch is reviewed before the next begins. Errors caught in batch N don't propagate to batch N+1.

5. **The excretory system** (IRF-LIQ-003) — three-pass filtration that removes dead signals, isolated formations, stale repos, classification drift, and governance debt. This runs BEFORE distillation begins, reducing the input set.
