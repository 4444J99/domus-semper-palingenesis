# Research-Path Design — Layer-Escalating Resolution for Cascade Residue

**Generated:** 2026-05-01 (this session)
**Premise:** For every unresolved item from the parity-rectification cascade, design a research path that climbs the hierarchy of abstraction one layer at a time. Stop only when the answer is either definitive at that layer, or the layer reveals a mathematical impossibility (information-theoretic, decidability, or referential).

**Three impossibility classes used as terminators:**
- **I-T (information-theoretic)**: the data needed to answer was never recorded; recovery requires an oracle
- **D (decidability)**: analog of the halting problem; content semantics cannot be decided without execution or external comparison
- **R (referential)**: answer depends on a prior held only by the user (intent, preference, policy); no climb can extract it

## Mathematical impossibility frontier (where escalation cannot continue)

For completeness, the points where the climb *must* stop because the structure of the question forbids ascent:

1. **Information-theoretic (I-T)**: a file's bytes existed only in this clone; no backup; no remote. *Recovery is impossible by physics of information storage.* The escalation cannot ask a higher-layer question because there is no higher layer of recoverable state.

2. **Decidability (D)**: deciding whether a stash should be popped requires modeling the user's intent at stash-time, which is not extractable from the stash itself. Climbing higher gives "what's the user's general intent for stashes?" which collapses to user policy (R). *Decidability bound is reducible to R.*

3. **Referential (R)**: the answer is held only in the user's value system; no climb can extract it because the value system is the climb's terminus. *The hierarchy bottoms out in the user.* This is the final mathematical wall — every system has a referent that must be axiomatically given.

The escalation succeeds — *every* unresolved item climbs to a terminator that is either (a) automatable, (b) a small bounded inspection, or (c) a single referential decision from the user. **No item is intractable; the cascade has a definite finish line.**
