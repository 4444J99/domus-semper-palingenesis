# Plan: Variablize All Names and Numbers in system-system--system

## Context

The system-system--system ontological research repo has hardcoded names, numbers, versions, and vocabulary terms scattered across TOML manifests, YAML registries, coverage reports, and the `piece` CLI. When atom counts change, category sizes shift, or version strings bump, these values must be updated manually in multiple files. The goal: **everything is a variable** — computed from source data or defined in one place — so statistics stay dynamic and instance-name changes propagate automatically.

## Architecture: Three Layers

| Layer | File | Role |
|-------|------|------|
| **Source of truth** | `sys.toml` (NEW) | Identity constants, versions, schemas, vocabulary |
| **Computation engine** | `piece` CLI (EXTENDED) | `--stats`, `--export-env`, `--generate-coverage` |
| **Shell integration** | `pieces.zsh` (EXTENDED) | `eval $(./piece --export-env)` loads all `SYS_*` vars |

### 2. MODIFY `piece` CLI — add `load_sys_config()` + three new modes

**a) `load_sys_config()`** — reads `sys.toml` from manifest root, returns dict with fallback defaults

**b) Replace hardcoded constants:**
- `merge_scale_dir` line 72: `"0.2.0"` / `"piece/v2"` → read from `sys.toml`
- `emit_graph` line 279: `"Helvetica"` / `10` → read from `sys.toml [config]`
- `telescope` line 243: `max_depth=10` → sentinel `None`, lookup from `sys.toml`
- `main` line 385: indent `2` → read from `sys.toml [config].json_indent`

**c) `--stats` mode** — compute all statistics from source data:
- Parse `atom-registry.yaml` (requires PyYAML, conditional import)
- Count atoms total, by category (ATM-X prefix), by domain, nature, formalization, provenance, layer
- Count laws from `LAWS.toml`, pieces from each manifest, documents from YAML
- Count integration phase assignments
- Output as JSON

**d) `--export-env` mode** — emit shell `export` statements:
- Combine `sys.toml` constants + computed stats into `SYS_*` env vars
- Naming: `SYS_VERSION_MANIFEST`, `SYS_ATOM_TOTAL`, `SYS_ATOM_S`, `SYS_LAYER_COUNT`, etc.

**e) `--generate-coverage` mode** — regenerate `coverage/*.md` from source data:
- Templates embedded in CLI, parameterized by computed stats
- Generates all 7 files: by-domain, by-layer, by-nature, by-formalization, by-provenance, integration-priorities, proof
- Each file gets `<!-- GENERATED by ./piece --generate-coverage -->` header
- Observations are templated with computed numbers (formulaic assertions) + static analytical notes

**f) Enhance `--validate`:**
- Check all manifest `version`/`schema` fields match `sys.toml`
- Check `piece_count` values match actual sub-manifest counts
- Check `LAWS.toml` `count` matches actual law entries
- Check `atom-registry.yaml` `total_atoms` matches computed count
- Check nature vocabulary in atoms matches `sys.toml [vocabulary.nature].terms`

### 3. MODIFY `pieces.zsh` — add `SYS_CONFIG` + eval block

```zsh
export SYS_CONFIG="${SYS_CONFIG:-${0:a:h}/sys.toml}"

# Load system identity & computed statistics (guarded)
if command -v python3 &>/dev/null && [[ -x "${0:a:h}/piece" ]]; then
    eval "$("${0:a:h}/piece" --export-env 2>/dev/null)"
fi
```

Existing `P__*` declarations unchanged. The eval is guarded — fails silently if `piece` unavailable.

## Variable Naming Convention

| Prefix | Category | Examples |
|--------|----------|---------|
| `SYS_VERSION_*` | Versions | `SYS_VERSION_MANIFEST`, `SYS_VERSION_LAWS` |
| `SYS_SCHEMA_*` | Schemas | `SYS_SCHEMA_PIECE`, `SYS_SCHEMA_LAW` |
| `SYS_SOP_*` | SOP IDs | `SYS_SOP_PRIMARY` |
| `SYS_LAYER_*` | Layers | `SYS_LAYERS`, `SYS_LAYER_COUNT` |
| `SYS_ATOM_*` | Atom stats | `SYS_ATOM_TOTAL`, `SYS_ATOM_S` ... `SYS_ATOM_V` |
| `SYS_PIECE_COUNT_*` | Piece counts | `SYS_PIECE_COUNT_S0` |
| `SYS_LAW_*` | Law metadata | `SYS_LAW_COUNT` |
| `SYS_DOC_*` | Document stats | `SYS_DOC_COUNT` |
| `SYS_INTEGRATION_*` | Phase counts | `SYS_INTEGRATION_1A` |
| `SYS_NATURE_*` | Nature vocab | `SYS_NATURE_VOCAB_SIZE` |
| `P__*` / `PN1__*` / `P1__*` | Pieces (unchanged) | `P__TOKEN__NAME` |

## What Stays Literal (and Why)

- **Prose in LAWS.md, axioms/, derivations/**: Documentation, not configuration. References to "7 layers" in prose explain the system; templating them destroys readability.
- **Atom IDs** (ATM-S-001): Stable identifiers, not config.
- **Date strings** (2026-04-04): Historical provenance, not dynamic.
- **PIECE_VAR_RE regex**: Structural invariant of the naming convention.
- **TOML version literals**: TOML has no interpolation. Values stay literal but are validated against `sys.toml`.
