## Pending — user decision required

### Upgrade pathway: Homebrew vs npm

`brew upgrade gemini-cli` returned `Warning: gemini-cli 0.38.2 already installed`. Homebrew's formula is at 0.38.2; npm's `@google/gemini-cli` is at 0.40.0. Confirmed:

```
brew info gemini-cli  →  stable 0.38.2 (bottled)
npm view @google/gemini-cli version  →  0.40.0
```

This is a structural lag, not a fixable error. Two options:

**Option A — Wait for Homebrew (passive).** The formula typically updates within 3–14 days of upstream. Stay on 0.38.2 until then. The config edits above mitigate the worst symptoms; the streaming-hang fix from 0.39.0 won't apply, but pinning the model and disabling bulk auto-load eliminates the practical hang trigger.

**Option B — Switch to npm (active).** Run:

```bash
brew uninstall gemini-cli
npm install -g @google/gemini-cli
```

Implications:
- Future upgrades: `npm install -g @google/gemini-cli@latest` instead of `brew upgrade`.
- `~/.config/homebrew/Brewfile` should remove the `gemini-cli` line so `brew bundle` does not reinstall it.
- This trades Homebrew's predictable lag for npm's bleeding-edge but un-bottled distribution.

**No default chosen.** This is a tooling-ownership decision the user should make consciously. Not auto-executing.
