# 01 — Context

## The trigger

Two Gemini CLI sessions, back to back. The first hung in "Thinking..." for **13m 44s** mid-response after silently switching from `gemini-3.1-pro-preview` to `gemini-3-flash-preview`. The second session burned **360,642 input tokens for a 52-token reply** with **0 cache reads** on a `/stats` no-op. Two further sessions confirmed the pattern: short prompts also hang when the auto-router selects `gemini-3-flash-preview`, which returns 0/0/0 on every call from this OAuth tier.

## Surface diagnosis

Three independent failure modes converge:

1. **Context bloat.** 106 `GEMINI.md` files in `~/Workspace`, totaling 1,126,786 bytes (~280K tokens). Settings `context.loadMemoryFromIncludeDirectories: true` + `context.fileFiltering.respectGitIgnore: false` recurse through every workspace subfolder, even gitignored content. With `Cache Reads: 0`, the cost recurs every prompt.
2. **Auto-router fallback hang.** `model.name: "auto-gemini-3"` lets the wrapper attempt mid-stream fallback from `gemini-3.1-pro-preview` to `gemini-3-flash-preview`. The fallback model is unreachable on this OAuth tier (4 sessions of empirical evidence: 0 input, 0 output, 0 cache reads). The wrapper does not surface the error; it hangs.
3. **Two minor versions behind.** Installed 0.38.2; npm latest 0.40.0. Upstream changelog confirms relevant fixes: 0.39.0 added "retry additional OpenSSL 3.x SSL errors during streaming"; 0.40.0 fixed "non-string model flags in resolution." Homebrew formula is still at 0.38.2 — see `02-actions.md` for the upgrade-pathway decision.

## The architectural reframe

The user observed: every other layer of the system uses atoms (function atoms, ATM-backlog atoms, in-session task atoms). Memory was the missing layer. A 17 KB `GEMINI.md` is a monolith that loads in full whether the prompt needs 2 lines of it or 200, and the existence of 106 such files turns session startup into a 280K-token tax.

Re-reading the largest `GEMINI.md` confirms a sharper diagnosis: **the bloat is machine-emitted**, not human-authored. The `<!-- ORGANVM:AUTO:START -->` / `<!-- ORGANVM:AUTO:END -->` block (~14 KB of the 17.4 KB file) is generated by the `organvm` tool. Hand-written context (~3 KB) is small and load-bearing. The auto-generator is the source of the pathology. Fixing the bloat means fixing the generator, per the existing axiom *"fix bases, not outputs."*

## What this means

- **Short term (this session):** flip the two settings, pin the model, get Gemini usable on 0.38.2.
- **Medium term:** refactor `organvm` to emit atoms instead of a monolithic auto-block.
- **Long term:** every `GEMINI.md` / `CLAUDE.md` / `AGENTS.md` migrates to manifest+atoms, opportunistically as repos are touched.

The architecture for this is in `architecture/ATOMIC-MEMORY.md`. The actions taken in this session are in `02-actions.md`.
