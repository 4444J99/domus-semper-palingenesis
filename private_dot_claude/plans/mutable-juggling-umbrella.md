## Implementation Plan

### Phase 1: Produce the Inventory Document
Write a complete `ORIGIN-DOCUMENT-INVENTORY.md` at the workspace root (or in `orchestration-start-here/`) containing:
- Every file path organized by the 16 categories above
- Word count per file
- Date modified per file
- Frontmatter presence (Y/N)
- Classification tag

**Method:** Python script that walks both trees, reads metadata, classifies files, and outputs structured markdown.

### Phase 2: Classification Pass
Automated classification using file path patterns + content signals:
- `plan` — files in `.claude/plans/`, `.gemini/plans/`, `.codex/plans/`
- `post-flood` — files in `post-flood/`
- `session` — files in `sessions/`
- `dialogue` — files in `prompt-corpus/dialogues/`
- `research` — files in `research/`
- `governance` — files containing COMMANDMENTS, LOGIC_FRAMEWORK, constitution
- `spec` — files in `specs/`, `specifications/`, `schema-definitions/`
- `content` — files in `content-pipeline/`
- `corpus` — files in `corpus/`, `testament/`
- `voice` — files in `worksheets/`, `constitution/`
- `doc` — files in `docs/`
- `memory` — files in `.claude/memory/`
- `intake` — files in `intake/`
- `archive` — files in `bench/`, `archive/`

### Phase 3: Atomization Readiness Report
For each category, output:
- Total file count
- Estimated atom count (1:1 for already-atomic, 1:N for large session files)
- Blockers (needs triage, needs decomposition, needs deduplication)
- Recommended processing order

### Verification
- Cross-check file counts against this plan's numbers
- Spot-check 5 files per category for correct classification
- Validate no origin material was missed by checking directory listings against inventory
