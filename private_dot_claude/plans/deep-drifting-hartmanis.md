# Plan: Prompt-Atom Decomposer — Smallest Atomic Unit with Clear I/O and Hierarchical DNA

## Context

The prompt corpus at `meta-organvm/praxis-perpetua/prompt-corpus/` contains 909 clean prompts in 535 sequences (LAST-WEEK-CLEAN.md). The ORGANVM system already has a formal atoms module (`organvm-engine/atoms/`) that decomposes *plans* into task atoms with a 13-field JSONL schema. The prompt corpus captures *raw human intent* — the layer below plans.

The user wants to decompose each prompt to its **smallest possible atomic unit** (a directive verb phrase: verb + object + context) with:
- **Clear I/O** per atom (input trigger, output product)
- **Hierarchical DNA** classified by the 9-phase design grammar (DEFINE→EXPAND→VARIABLE→RIGOR→PILLAR→SOURCE→PARAMETER→NEXT→TRANSFORM)
- **Relationship tracking** across 6 hierarchy levels

The existing QA test suite (136 tests, all green) was completed in the prior session and is preserved.

## 6-Level Hierarchy

```
L0  Organ         META, ORGAN-I, ..., AXIOM, LIMINAL, UNCLASSIFIED, SEED
L1  Project       Workspace path (org/repo from ### header)
L2  Sequence      One session burst (### block, 1-535)
L3  Prompt        One timestamped utterance (`HH:MM:SS`)
L4  Directive     One verb-phrase unit — THE ATOM
L5  Entity        Named objects: file paths, $VARs, URLs, quoted concepts
```

## Prompt-Atom JSONL Schema

```python
{
    "id": str,           # sha256[:12] of (seq_num, timestamp, directive_index, text[:80])
    "raw_text": str,     # exact substring this atom was extracted from

    "hierarchy": {
        "L0_organ": str,       # "META", "ORGAN-III", etc.
        "L1_project": str,     # workspace path from ### header
        "L2_sequence": int,    # seq_num (1-535)
        "L3_prompt": str,      # prompt_id: sha256[:12] of (seq_num, timestamp, text[:100])
        "L4_directive": int,   # 0-based index within prompt
        "L5_entities": list,   # file paths, $VARs, URLs, quoted concepts
    },

    "parent_ids": {
        "sequence_id": str,    # "seq:{seq_num}"
        "prompt_id": str,      # sha256[:12] of containing prompt
    },

    "dna": {
        "primary_phase": str,      # DEFINE|EXPAND|VARIABLE|RIGOR|PILLAR|SOURCE|PARAMETER|NEXT|TRANSFORM|NONE
        "secondary_phases": list,  # additional matching phases
        "confidence": float,       # 0.0-1.0
        "mechanism": str,          # digestive|skeletal|nervous|circulatory|integumentary|""
    },

    "io": {
        "input": str,       # what triggered/feeds this atom
        "output": str,      # what this atom demands as product
        "io_type": str,     # imperative|interrogative|declarative|relay|context_setting
    },

    "signals": {
        "verb": str,             # imperative verb (or "")
        "object": str,           # direct object
        "context_clause": str,   # remaining context
        "mentions_files": list,  # file path references
        "mentions_vars": list,   # $VARIABLE references
        "quoted_content": bool,  # contains embedded quotes
    },

    "source": {
        "date": str,        # 2026-03-28
        "time": str,        # 00:07
        "timestamp": str,   # 00:07:38
    },

    "relationships": {
        "depends_on": list,      # atom IDs this one logically requires
        "sibling_ids": list,     # other atoms from the same L3 prompt
        "related_seeds": list,   # seed_types if prompt was classified as a seed
    },

    "tags": list,               # technology/domain tags
}
```

## Core Algorithm: 5 Prompt Structure Types

Before splitting, classify each prompt to determine the splitting strategy:

| Type | Trigger | Strategy | Example |
|------|---------|----------|---------|
| ATOMIC | `<80 chars`, no `;` or conjunctions | Pass through as 1 atom | "proceed", "yes", "what is next?" |
| COMPOUND | Contains `;` joining independent clauses | Split on `;` | "commit to remote; research the vacuum; log it" |
| RELAY | Embedded quotes, "message from:", leading `"` | Frame directive = atom, quoted = L5 entities | "message from maddie: 'I love it!!'" |
| ENUMERATIVE | `1.`/`2.` numbered items | Each item = 1 atom | "1. create schema\n2. write tests\n3. deploy" |
| STREAM | Long multi-sentence, no structural delimiters | Sentence split → merge verb-less sentences back | Long design monologue |

## DNA Classification: 9 Design-Grammar Phases

From `library/chains/design-grammar.yaml`. Each atom is scored against all 9 phases:

| Phase | Mechanism | Recognition Verbs | Recognition Patterns |
|-------|-----------|-------------------|---------------------|
| DEFINE | digestive | define, declare, specify, formalize | imperative + specific technical object |
| EXPAND | digestive | expand, extend, scale, broaden | scope widening, system-wide, orchestration |
| VARIABLE | skeletal | name, label, assign, bind | `$UPPERCASE_SNAKE_CASE`, naming acts |
| RIGOR | skeletal | cite, reference, ground | peer-reviewed, canonical, scholarly |
| PILLAR | skeletal | decompose, separate, partition | `$PILLAR_*`, numbered constraint domains |
| SOURCE | skeletal | cite, source, evidence | exact papers, 30-80 sources, DOI/ISBN |
| PARAMETER | nervous | set, configure, toggle | `$DEPTH`, lean MVP vs full-stack |
| NEXT | circulatory | produce, deliver, build | `$NEXT`, next artifact, build exactly |
| TRANSFORM | integumentary | convert, transform, tighten | pitch deck, one-pager, landing page |

**NONE** when no phase matches. Highest-scoring → `primary_phase`; others → `secondary_phases`.

## I/O Extraction Rules

| io_type | Input | Output |
|---------|-------|--------|
| imperative | referenced artifacts/state or "session context" | product named by verb+object, refined by DNA phase |
| interrogative | question's domain context | "answer: {question focus}" |
| declarative | "observation context" | "context_established: {subject}" |
| relay | "external message from {source}" | "acknowledgment + routing decision" |
| context_setting | "background information" | "context_established: session framing" |

Phase-specific output refinement: DEFINE → "exact specification: {object}", VARIABLE → "computable binding: {$VAR}", TRANSFORM → "format transformation: {target}", etc.

## Key Design Decisions

1. **Prompt structure pre-classification** before splitting avoids a brittle one-size-fits-all sentence splitter on informal, stream-of-consciousness text
2. **ATOMIC passthrough** for short prompts preserves "yes", "proceed", "all of the above" as single atoms with `dna.primary_phase=NONE`
3. **RELAY detection** tags embedded cross-session quotes (messages from Maddie, Gemini output, pasted transcripts) — the human's framing directive is the atom, quoted content lives in L5 entities
4. **Multi-phase DNA** allows a single atom to carry DEFINE + VARIABLE when it both defines a structure and names it with `$ENV` notation
5. **No external dependencies** — all stdlib (re, json, hashlib, pathlib, collections)

## Expected Output

- **~1,200-1,500 atoms** from 909 prompts (expansion ratio ~1.3-1.6x)
- **~30-40% NONE phase** (confirmations, questions, informal speech)
- **~15-20% DEFINE/EXPAND** (imperative design directives)
- **~5-8% RELAY** (cross-session pastes, partner messages)
