Never close GitHub issues unless the user explicitly says to close them. "Address" / "remediate" / "requiring attention" means advance the work — implement, fix, move the ball forward. It does NOT mean triage, categorize, or close.

**Why:** User was very clear that closing issues without permission is unacceptable. Issues represent work to be done, not items to be swept away.

**How to apply:** When the user says issues "require addressing" — read them, understand them, and start implementing solutions. Ask for priority order if there are too many to tackle at once. Only close an issue when the user explicitly says to close it or when you've shipped the fix and the user confirms.
