Three operational rendering modes, each a different language describing the same phenomena:

1. **Logic** — qualification/disqualification proofs. Gate conditions as propositions, CHECK predicates as pass/fail, status fields as truth values. Format: YAML. The organism proves its own existence through accumulated qualifications.

2. **Math** — numbers for measuring, tracking, formalizing. Observation summaries, route counts, attraction counts, defect inventories, variance deltas. Format: JSONL. Universal language between sessions/agents — requires no philosophical context to interpret.

3. **Biology** — organic abstractions mirrored as algorithmic functions. Function names ARE the natural phenomena they harness. Creation/destruction = same universal force in different phase. Format: Python. The organism embodies natural processes, not metaphors for them.

**Why:** The operator identified these as the three ways existence is rendered — through constructed logical language, through numerical proof, and through biological/alchemical phase-change. The organism already produces all three but hadn't named them as a unified triad.

**How to apply:**
- Every capability has three simultaneous representations: its gate contract (logic), its measurements (math), its function (biology). All three describe the same referent.
- Forward phase: chaos-abstraction → logic-parsing → math-grounding → biology-embodiment
- Backward phase: function-output → math-measurement → logic-evaluation → biology names next hunger
- When building new functions, ensure all three representations are created: the contract (logic), the test assertions (math), and the function (biology).
