---
name: Additive design, never reductive
description: Each user prompt adds complexity to the design — never simplify, reduce, or replace prior layers
type: feedback
---

Each user prompt is an addition rolled into a continually complicating and elevating end system, not a reduction or replacement of prior work.

**Why:** The user builds systems through layered accumulation. A tree diagram doesn't replace the YAML schema — it adds a visual layer on top. A triad-per-level doesn't replace the three-tier model — it refines it into five levels while keeping all prior structure.

**How to apply:** When presenting design iterations, explicitly show how each new element layers on top of everything before it. Never drop prior design sections when adding new ones. The final design is the sum of ALL sections, not the latest one.
