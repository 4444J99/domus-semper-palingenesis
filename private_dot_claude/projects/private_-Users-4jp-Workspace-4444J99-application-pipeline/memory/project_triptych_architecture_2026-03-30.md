**SOP-INST-001** ("The Forced Revision Protocol") was created in aerarium. 7 phases: Research (parallel agents) → Raw Material → Structure (forced compression) → Verification (hall-monitor) → Citrinitas (read-aloud purification) → Submission (human-gated) → Post-submission. This SOP needs to be adopted by the application pipeline — currently it generates cover letters and portal answers in one pass, violating the forced revision principle.

**Why:** Creative Capital was DEFERRED because the funder-fit gate failed (CC funds sensory art, ORGANVM produces infrastructure). This gate saved wasted effort. The application pipeline has NO equivalent funder-fit gate for jobs.

**How to apply:** The pipeline should implement the same principle: research before writing, forced revision before submission, verification as integrated cycle not post-hoc patch. The SOP's Phase 1 (parallel agent dispatch for research) maps directly to what the pipeline needs for contact research, role analysis, and funder-fit assessment.
