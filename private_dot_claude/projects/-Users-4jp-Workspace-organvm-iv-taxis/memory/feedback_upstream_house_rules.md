name: Upstream House Rules First
description: Always read CONTRIBUTING.md and project guidelines before posting comments or pushing code to upstream repos
type: feedback
---

Before acting on any upstream PR (comment, ping, code push, rebase), read the project's contribution guidelines first. We are guests.

**Why:** User correction — plan jumped to drafting comments and pushing code without checking each repo's CONTRIBUTING.md, CLA/DCO requirements, PR etiquette, CI expectations, or stale bot configuration. "Ensure that ALL best-practices and contribution-guidelines are followed by the guest's house we are kicking our feet up at."

**How to apply:** For every upstream contribution session, Phase 0 is always: fetch and summarize CONTRIBUTING.md, CODE_OF_CONDUCT.md, PR templates, CI config, and any CLA/DCO requirements. Incorporate their specific rules into comment drafts and code changes before executing.
