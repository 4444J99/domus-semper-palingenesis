Open-source contributions use the Cross-Organ Symbiote pattern (Approach B, designed S30):

**Why:** The contribution IS the process, not just a PR. Each contribution exercises 5 organs simultaneously, producing artifacts in each. The public process demonstrates ORGANVM working as designed.

**How to apply:**
- ORGAN-IV (Taxis): `contrib--{project}/` workspace — journal, seed.yaml, plans, fork
- ORGAN-I (Theoria): Theory extraction docs — formalized patterns from the fusion
- ORGAN-II (Poiesis): Visualization artifacts — Mermaid diagrams, before/after
- ORGAN-III (Ergon): Shipped code — the PR payload, idiomatic to target codebase
- ORGAN-V (Logos): Public narrative — essay telling the contribution story

Lifecycle: JOURNAL → EXTRACT → GENERATE → BUILD → SHIP → NARRATE (cycle, not pipeline)

The code must be native to THEIR codebase — ORGANVM concepts, their idiom. Forward-only lifecycle. seed.yaml required. Journal every session.

**Portable system prompt:** `contrib--adenhq-hive/OPEN-SOURCE-CONTRIBUTION-SYSTEM-PROMPT.md` — drop into any new session to initialize the machine for a new target.

**First instantiation:** AdenHQ/Hive (S30). PR #6707, 5 organs delivered.
**Second+ instantiations:** Managed by S32 "plague campaign" — anthropic-skills, temporal, dbt-mcp, langchain, primeinc, ipqwery.
