New concept from 2026-04-05 session. Instead of "dissolved, deleted, or absorbed," every dissolving repo undergoes archaeological decomposition into primitives before the directory can rest:

1. **Identify primitives** — reusable atoms inside the repo (schemas, patterns, algorithms, governance formalisms)
2. **Catalogue** — what each primitive is, why it's valuable, what it depends on
3. **Extract** — de-couple from macro context (the repo)
4. **Place** — re-couple to destination (gate contract, dna entry, system library atom)
5. **Verify the bond** — primitive in new context still means what it meant

**Sequential containment** (user preference, 2026-04-05): Process repos one at a time, complete each fully before the next. Transfer learning compounds — each decomposition sharpens the tool for the next. Primitive collisions detected at extraction time, not after. Then full 8-organ dialogue composes all primitives.
