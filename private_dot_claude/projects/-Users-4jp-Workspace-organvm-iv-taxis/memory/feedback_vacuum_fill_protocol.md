When running the 10-index propagation checklist at session close, N/A is not acceptable. Every "not applicable" is a vacuum where something should exist.

**Why:** The user explicitly called this out in S30: "all the N/As suggest something imperative; it means there is a vacuum where something should be — research it, plan it, log it." N/A means you didn't look hard enough.

**How to apply:**
1. For each index marked N/A, ask: "Does this session's work create ANY entity, event, or state change that this index tracks?"
2. If yes: fill it. Create the issue, log the evidence, update the scorecard.
3. If genuinely no: explain WHY in one sentence (e.g., "No SGO research — session was pure infrastructure"). A reasoned skip is different from a lazy N/A.
4. The default is check-all-10 and consciously skip inapplicable ones, not check-none.

Examples from S30:
- Omega #12 was marked N/A but PR #6707 is the FIRST external contribution (1/3)
- Inquiry log was marked N/A but the theory extraction is evidence for INQ-2026-002
- GitHub issues were marked N/A but the contrib engine needed Phase 2 tracking issues
