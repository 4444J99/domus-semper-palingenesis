Portals exist where rendered reality and imagined ideal diverge. They open at the front line — where dystopia is gaining mass — inject the minimum form to shift the trajectory toward utopia, then precipitate the next intervention and stop.

**Why:** The operator manages parallel sessions across multiple agents. Each portal is a self-contained intervention that deposits its forward state for the next portal. The avalanche continues only when the operator picks up the baton. No awkward walk-in (the BRIEFING is the war report). No awkward exit (RECEIPT.md IS the baton for the next runner).

**How to apply:** When the operator describes a mismatch between what is and what should be at any scale, open a portal. When you see a `tasks/` directory, it contains active interventions. When you see a `RECEIPT.md`, the previous portal completed and deposited forward state.

## The 5-File Genome

| File | Function | Scale | When written |
|------|----------|-------|-------------|
| `CLAUDE.md` | Portal nature + counter-force field + lifecycle | 4 (module identity) | At creation |
| `BRIEFING.md` | War report: front line, attractors, deliverables | 1-2 (matter-prime → instantiation) | At creation OR by previous portal's PRECIPITATE |
| `REFERENCES.md` | Keys only — doors + context stream + terrain | 6 (communal acknowledgment) | At creation |
| `SEQUENCE.md` | Intervention procedure α→ε | 3-5 (function → sequence) | At creation |
| `RECEIPT.md` | Forward deposit — baton for next portal | **Phase transition** | At completion (PRECIPITATE) |

## Lifecycle

```
INJECT → CREATE → OPERATE → VERIFY → PRECIPITATE → STOP
```

- **No EVAPORATE phase.** Evaporation is the next portal's prerogative (or the operator's).
- **PRECIPITATE** writes RECEIPT.md + optionally seeds next portal's BRIEFING.md.
- **STOP** is absolute. The agent does not continue. The operator reads the receipt and decides.

## The Avalanche

```
PORTAL₁ executes → PRECIPITATE → RECEIPT.md + next BRIEFING.md → STOPS
operator reads receipt → cd tasks/<next-slug>/ → claude → PORTAL₂ reads BRIEFING
PORTAL₂ executes → PRECIPITATE → RECEIPT₂.md + next BRIEFING.md → STOPS
(and so on, until the operator stops picking up batons)
```

The avalanche has infinite forward motion and natural stopping points. Each stop is a receipt. Each continuation is the operator's choice.

## The Scale Ladder

| Scale | Name | Portal expression |
|-------|------|-------------------|
| 1 | **Matter-prime** | The operator's raw prompt, preserved verbatim |
| 2 | **Action-instantiation** | Ledger entry recording portal creation |
| 3 | **Functional-function** | Atomic procedures in SEQUENCE.md with dependency edges |
| 4 | **Modular-function-relays** | The 5-file genome itself |
| 5 | **Sequence-modules** | SEQUENCE.md relay circuit α→ε |
| 6 | **System-communal** | REFERENCES.md + BRIEFING.md — recognize existing, arrange into delivery |

**Critical principle at scale 6:** The system doesn't build from scratch. It RECOGNIZES what already exists, then ARRANGES it. The portal's job is midwifery, not genesis.
