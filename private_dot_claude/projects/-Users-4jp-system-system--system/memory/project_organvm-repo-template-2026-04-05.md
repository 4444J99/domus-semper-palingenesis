## Context (started 2026-04-05, paused for dream consolidation)

The user is designing a universal repo template for all 105+ ORGANVM repos. The vision:
- Every project needs: research (past/present/future), theory of code, philosophy of domains
- Research is always the same TYPE of forward motion — "bring back specified information for that node"
- Merge all domains (market analysis, governance, automations, public process) so they are "exactly what they're meant to be"
- Only create directories when "absolutely necessary to be nested" and "100% unique to the thing it is trying to be"
- All documents in Markdown, all text in one location
- Theory/documentation/research → processed as atomic units → implementable, testable, improvable
- The name concept: "universal modular fractal alchemical synthesizer"

**Why:** The user sees repos as materons — each is a reconfigurable space where theory becomes matter. The formal ontology from system-system--system should be the generative pattern for all repos.

**Design factor (2026-04-05):** "theoretical changes iterate new-instances as meta-theoretical non-destructive spaces for creation and destruction" — when theory changes, it forks a new instance-space rather than overwriting. Old theory persists. Maps to multiverse pipeline, append-only registries, git branching.

### Empirical survey (12 repos across organs)
**Universal skeleton** (80%+ repos): README.md, seed.yaml, ecosystem.yaml, CLAUDE.md, AGENTS.md, GEMINI.md, docs/, ecosystem/ (always: intelligence/, pillar-dna/, snapshots/)
**Conditional** (40-60%): src/, tests/, pyproject.toml or package.json, scripts/, data/, CHANGELOG.md
**Unique-to-project** (≤2 repos): bible/, museum/, drafts/, atlas/, engine/, brahma/, corpus/, prompts/, strategy/, experiments/

**Key insight:** ecosystem/governance infrastructure is universal; domain-specific content directories are unique. The boundary between generic and project-specific needs to be discovered from the data, not imposed from a menu.

### Status
- User chose "universal repo template" as first deliverable (over formalization or combined)
- User rejected multiple-choice question about nesting rules — wants to "discover the rules binding unique project info and generic"
- Brainstorming is paused, not abandoned. Next step: propose approaches based on empirical findings.
