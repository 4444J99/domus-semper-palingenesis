---
name: Address root causes not instances
description: When correcting defects, always amend the generating process/protocol, not just the individual outputs
type: feedback
---

When a review surfaces multiple defects from the same process, the cascade must address the root — the structural property of the process that generated the defects — not just the individual instances.

**Why:** User explicitly flagged this when approving a cascade correction plan that fixed 6 atom-registry defects but didn't amend the SOP that produced them. Instance-level fixes recur on next application.

**How to apply:** Every correction plan for process outputs must include a protocol amendment step that modifies the generating process itself. The amendment should be a formal artifact (not just a note) that future applications of the process will consume.
