Never mark an external index propagation item as "N/A." Every item must be researched and get a definitive verdict: Done, Deferred (with reason + plan), or Planned (with construction plan).

**Why:** The user considers N/A lazy and unacceptable. A vacuum where something should be is not "not applicable" — it's an unresearched gap. Even genuinely inapplicable items need a one-sentence justification for WHY they're inapplicable, not a bare "N/A."

**How to apply:** When completing the 10-index propagation checklist at session close, check every single index against the session's work. If blocked, write the plan that unblocks it. If deferred, document exactly what needs doing and when. If truly inapplicable, state the specific reason (e.g., "No SGO research commissions touched — all work was engineering").
