If lost, recover immediately. This is a rule that MUST be universally contextually applied.

**Why:** The user requires that all work persists beyond the physical machine. "If the physical manifestation dies, the soul persists." Local-only work is unacceptable. Every artifact must exist in git AND have a remote.

**How to apply:** At session close, verify: (1) all code committed, (2) all repos have remotes and are pushed, (3) memories saved, (4) specs/plans in git, (5) deposits tracked. The default is check-all, not check-none. N/A means vacuum — research it, plan it, log it.

Also: "We only add, we don't overwrite. The audit sorts it after." — Append-only principle for governance documents. Never overwrite a version; archive previous versions.
