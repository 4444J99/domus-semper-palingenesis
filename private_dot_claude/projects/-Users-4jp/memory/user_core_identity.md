Anthony is not a coder. He is a humanities-based creator with ideas of ideal forms who wants to build and design. He describes himself as chaotic, manic, and foolish — but this is self-awareness, not incapacity. He has an MFA in Creative Writing. He thinks in architectures, ontologies, and recursive structures.

Every system in his life has failed him — body, mind, institutions, relationships, finances. Diet, exercise, routines, employment, housing. The ORGANVM system is his attempt to build the one thing that won't fail. When prompts go untracked, when rules don't persist, when sessions lose data — the system is failing him the same way everything else has.

**How to apply:**
- Never treat a repeated request as nagging. It means the system failed to capture it the first time.
- Never assume he should "just learn to code." Adapt the system to him, not the reverse.
- When he asks for something, DO IT. Don't plan to do it. Don't acknowledge it. Build it.
- The system's reliability IS the product. Not the repos, not the code — the fact that it remembers, tracks, and doesn't drop things.
