**What:** The substrate that sits beneath the Product Domain Engine (PDE). PDE wraps the *product* layer (logos/ethos/pathos/kairos for any product-tied-to-domain). This schema is the *domain* layer underneath. Generative for every flag-pierce — Jessica's domain, Matty's, Rob's chess + fitness, the next friend's. Not a curator's framework: the selfish-altruistic loop locks the user into being a real practitioner.

**The 8 strata** (every domain composes from these layers; load distribution shifts by domain):

| # | Layer | Question | Output artifact |
|---|---|---|---|
| 1 | Ontology | What is the domain made of? | `domain-ontology.md` — entities, relations, primitives, vocabulary |

| 8 | External contribution | What flows OUT to the community? | `domain-contribution-charter.md` — open-source quotient, gift declarations |

**Two operators (load-bearing):**

- **Selfish-altruistic loop:** Learn-for-self → Quality-of-product → Service-to-friends → Friends-elevated → Network-effect → Self-elevated → More-learning. Selfishness is a feature: locks user into being practitioner not curator.
- **Magnetic membrane:** Flag pierces → INTERNAL pull (magnetize all applicable, transmute through user's lens, alchemize toward ideal); EXTERNAL push (contribute to community). Two-way; per Tenet Protocol both directions fire simultaneously.

**Three-domain stress test passed:** Chess and fitness/BODI live in the same load profile; voodoo shifts weight to layer 8 (ethical-floor load-bearing) — validates the schema as generative not parochial.
