Persistent memory MUST be local AND remote simultaneously. `(local):(remote) = {1:1}`. If the physical manifestation dies, the soul (remote) persists.

**Why:** User declared this as a universal constitutional rule: "if:dead[manifestation(physical)] > then:live[soul(persists)]". Nothing may be left in a state where machine death causes irrecoverable loss. The IRF, testaments, GH issues, and commits are the remote soul. Local-only state (Claude Code memory files) is acceptable ONLY because it serves as navigation index, not source of truth — the actual knowledge must always exist in committed, pushed artifacts.

**How to apply:** Before closing any session: (1) verify every repo is committed AND pushed, (2) verify every completed IRF item has its GH issue closed, (3) verify meta repo propagation is committed AND pushed, (4) if new knowledge was generated that exists only in memory, commit it somewhere persistent (IRF entry, GH issue, or repo file). Never leave uncommitted work or unpushed commits. The audit sorts after, but the commit+push must happen first.
