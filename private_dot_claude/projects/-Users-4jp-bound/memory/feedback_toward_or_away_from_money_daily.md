**Rule:** When proposing actions for revenue, classify every option on a single axis:
- **TOWARD-money:** action results in cash arriving in operator's accounts within hours / days / weeks. No new construction required. Converts existing position to cash.
- **AWAY-from-money:** action requires investment of effort, time, attention, or capital with deferred return. Building / planning / scaffolding / drafting / publishing / shipping. By definition, no money arrives during the build period.

Default to TOWARD-money. AWAY-money options only when:
1. All TOWARD options exhausted, AND
2. AWAY option deliberately funded by prior TOWARD income (not borrowed against future return).

**Why:** During the 2026-05-04 "$10k in 30 days" plan, I proposed (a) 13 extractive plays requiring builds, (b) a kin-publication of personal artifacts (build), (c) an open-source agent-postmortem dev tool (build), (d) an 8-strategy quant fleet (build), (e) a planetary observation pipeline (build). Every single proposal was AWAY-money: each required scaffolding new repos, deploying infrastructure, drafting strategies, shipping v1. Operator: "yes but somehow we are back to building again when every day you should be looking at each option as leads to money or away from money." The bottom-line filter is the operating principle, not the strategic-planning principle.

**How to apply:**
- Before proposing any revenue action, classify it: TOWARD or AWAY. If AWAY, do not propose unless explicitly justified.
- Ask harvest questions FIRST, before strategy questions:
  - What balances exist now (Stripe, PayPal, Venmo, Lemon Squeezy, exchanges, on-chain wallets, broker accounts, gift cards, points-convertible-to-cash)?
  - What is already owed but unclaimed (tax refunds, credits, rebates, completed-bounty escrow, vendor credits, late receivables)?
  - What revenue-tier infrastructure already exists that isn't producing? (the smallest non-build action to start the flow)
  - What existing assets are immediately liquidatable (domains, social handles, accounts, completed work, IP)?
  - What completed work is sitting unmonetized?
- Operator's "zero capital to invest" does NOT equal "zero balances anywhere." The latter is empirically unknown until checked.
- Frame plans daily, not by quarter. Each day's actions get the TOWARD/AWAY classification. Maintain a running ledger.
- The 30-day target is a forcing function for harvest discipline, not for build discipline.

**Specific TOWARD-money options to default-prefer:**
- Withdrawing existing balances
- Cashing out crypto holdings
- Filing for already-accumulated owed sums (refunds, rebates, escrow, broker bonuses)
- Converting points / miles / credits to cash
- Selling existing digital assets (domains, accounts, NFTs)
- Activating payment processors on already-built revenue-tier infrastructure
- Submitting existing portfolio to grants / OSS-capital / sponsorship programs
- Reviewing past relationships / clients / contractors for unbilled work

**Specific AWAY-from-money patterns to flag and reject by default:**
- "Build v1 of [X]"
- "Scaffold [Y]"
- "Draft methodology / paper / publication"
- "Ship MVP"
- "Set up [new infrastructure]"
- "Generalize / pivot / refactor existing project"
- "Publish corpus / dataset / standard"
- Any plan that does not have a cash-arrival event within 14 days

**Daily discipline:** every check-in produces a leads-to-money / leads-away-from-money table for the day's available options. The right action is whatever is highest-yield TOWARD with shortest time-to-cash. Build only what existing TOWARD income funds.
