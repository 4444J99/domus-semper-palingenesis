**Rule:** When proposing revenue / strategic plays, the dominant filter is "is this shape legible to capital allocators?" If it isn't pattern-matched to a comp set with public valuations, recognized funding category, and clear scaling trajectory, reject it. Personal methodology, idiosyncratic kin publications, lifestyle businesses, founder-IS-the-product plays, pseudonymous internet hustles — all fail this filter even if they generate cashflow.

**Why:** During the 2026-05-04 "$10k in 30 days" plan I proposed (a) 13 extractive bounty/SEO/claim-mill plays, then (b) a kin-style integrated publication of operator's atom registry + voice constitution + multi-organ architecture as an open framework. Operator: "the problem with my personal work is it is personal--and besides the point--i want what is capital friendly above all else." The kin frame I'd just proposed STILL failed because operator's voice constitution / atom registry / eight-organ architecture are personal artifacts — they don't pattern-match to capital allocators' theses (capital doesn't acquire methodology papers; it acquires companies with TAM and revenue).

**How to apply:**
- Default shape for capital-friendly: **open-source-with-cloud-tier** (LangChain / ClickHouse / PostHog / Supabase / Cal.com pattern). This is the most-pattern-matched capital-friendly OSS shape and gets recognized instantly by AI/dev-infra investors.
- Generalize operator's domain expertise (Conductor, MCP, agent-fleet orchestration, voice/governance) AWAY from operator-specific into category-canonical. Don't ship "operator's Voice Constitution"; ship "Voice-as-Policy: declarative policy language for AI agents" with operator's voice constitution as the reference example, not the product.
- Comp set required: every play should name the public companies / funded startups it would be measured against (LangSmith, Braintrust, Humanloop, Galileo, Patronus, Vercel AI Gateway, Cloudflare Workers AI). If the play has no comp set, it's not capital-friendly.
- TAM ≥ $1B narrative: if you can't articulate a TAM ≥ $1B with a "why now" thesis, capital will not fund it.
- 12-18 month trajectory: capital-friendly plays generally have $0 revenue at 30 days. The first $10k often comes via angel check, not extraction. Operator's 30-day target conflicts with capital-friendly horizon; the conflict is structural, not solvable by cleverer extraction.
- Reject from default proposals: personal methodology papers, kin-style citation publications, pseudonymous bounty hunting, content/newsletter, lifestyle SaaS, claim mills, airdrop fleet farming, programmatic SEO empires. These are extractive or personal — not capital-friendly.

**Capital-friendly categories in 2026 (AI/dev-infra):**
- Agent runtime / fleet orchestration / MCP gateway
- AI eval / observability / governance platforms
- AI security tooling (CVE discovery automation, prompt-injection defense, agent supply-chain security)
- Vertical AI agents (legal review, code review, claim processing, customer service automation)
- AI-native data infrastructure (vector DBs, embedding management, retrieval)
- Agentic developer tooling (code-gen specialty, IDE extensions for AI workflows)

- "Publish operator's voice constitution as standard"
- "Atom corpus + methodology paper as kin reference"
- "Open-source the Conductor MCP as protocol specification (no commercial wedge)"
- Anything where operator's name / personal arc IS the product
