---
name: User design philosophy — one person as team, layered wedding cake, expand don't reduce
description: Core creative philosophy — builds in layers where each is a sellable artifact, the system IS the work, never cut scope as primary solution, expand with structure instead
type: user
---

**Identity**: One person acting as a team engaging in expansive inquiry. The portfolio IS the eight-organ system IS the proof IS the work. The 525 commits aren't portfolio-polishing — they're the engine producing output.

**The wedding cake model**: Build in layers. Each layer is a complete artifact — sellable, demonstrable, valuable on its own. The next layer adds depth, not replaces. The MVP is the project in totality — every layer is the MVP at that stage of completion.

**Expand, don't reduce**: When facing complexity, build more structure, not cut scope. Over-engineering is not vanity — it's one person compensating for not having a team. The infrastructure IS the differentiator. The depth IS the signal.

**Design sensibility**:
- **Density through layers** — all content exists but is hidden/revealed progressively
- **Anti-scroll** — long vertical pages lose attention. 74% of viewing time in first 2 screenfuls.
- **"Slick / sexy / sly"** — minimalism that contains weight. Layered, not sparse or cluttered.
- **Every element matters** — when the bento grid removed content, the user demanded it ALL back
- **No massive cards with three lines** — compact, dense cells
- **Click-to-expand, shift-the-grid** — the user explicitly described this bento interaction pattern

**Visual review required**: The user reviews the live site visually and gives direct feedback. Any visual change needs to be reviewed before considering it done.

**How to apply**: Never suggest cutting scope as a primary solution. When the roadmap feels large, layer it so each sub-phase ships independently — don't remove layers. Propose structure (sub-phases, dependency ordering, parallel tracks) instead of reduction. The contrarian's "stop building" thesis was rejected because the building IS the work.
