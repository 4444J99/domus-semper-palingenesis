name: Tracking Instruments
description: Memory, plans, and meta documentation are tracking instruments — not authoritative records. They observe, they don't define.

Memory, plans, and meta documentation are **tracking instruments**. They exist to watch the system move, not to be the system.

**Why:** The authoritative record lives in the triple-referenced locations (IRF, repo, GitHub issue). Memory files, plan files, and meta docs are indices — they point at the work, they track its state, they help future sessions find context. But they are not the identity of the work itself. A plan that never becomes code is a plan. A memory that never reaches an SOP is a note. Only the triangulated thing is real.

**How to apply:** When closing a session, the question is not "did I save to memory?" but "did the work land in its authoritative locations?" Memory is a convenience cache for agent continuity. Plans are draft scaffolding for implementation. Meta docs are observation layers. None of these substitute for the actual deliverable being committed, pushed, and triple-referenced. Don't mistake the tracking for the tracked.
