"A river is a function of natural necessity; we sit by it and feel its art — same thing needs apply here."

**Why:** The first function (skeletal_define.py) was over-declared: "Five capabilities," elaborate docstrings citing SEED procedures, named dataclasses with careful field docs. It read like a spec, not like a river. The specification is not the thing. The thing should feel inevitable — it exists because the organism needs it, and the art (temporal awareness, signal mapping, variance detection) emerges from that necessity, not from being named as features.

**How to apply:** Code should do what needs doing without announcing what it does. Functions speak for themselves. Docstrings say why, not what. Don't enumerate capabilities — let the caller discover them by reading. Don't name patterns — embody them. If a function reads the organism's structure and naturally records what it found, that's not "temporal recording (TRACE persistence per A3 + Law 3)" — that's just what looking IS. Strip declarative ceremony. Let necessity shape the code, and let the art be a byproduct.
