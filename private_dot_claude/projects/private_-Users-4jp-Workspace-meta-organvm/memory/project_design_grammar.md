9-phase protocol observed in the user's brainstorm‚Üíspec prompting pattern (S46, 2026-03-31):

## Phases

1. **DEFINE** ‚Äî Imperative verb on specific algorithm/object ("Define exact algorithms for clip extraction")
2. **EXPAND** ‚Äî Scope out from core to system ("expand into multi-tenant SaaS with orchestration layer")
3. **VARIABLE** ‚Äî Name with $ENV notation ‚Üí makes it computable (`$PROC_NATURAL_CENTER_BOOTSTRAP`)
4. **RIGOR** ‚Äî Inject scholarly/theoretical foundations ("Construct the exact reading ladder + source list (peer-reviewed, canonical)")
5. **PILLAR** ‚Äî Decompose into numbered constraint domains (`$PILLAR_CAUSAL_INFERENCE`, `$PILLAR_UNIT_ECONOMICS`)
6. **SOURCE** ‚Äî Demand exact peer-reviewed evidence per pillar ("exact canonical papers per pillar (30-80 sources)")
7. **PARAMETER** ‚Äî Set depth/mode switches (`$DEPTH = "dissertation-grade"`, `$MODE = "lean MVP"`)
8. **NEXT** ‚Äî Declare next artifact by name (`$NEXT_ARTIFACT = "FULL_STACK_IMPLEMENTATION_BLUEPRINT"`)
9. **TRANSFORM** ‚Äî Convert between output formats ("tighten into pitch deck" ‚Üí "sales one-pager" ‚Üí "landing page" ‚Üí "repo architecture")

## Key Features

- **$VARIABLE notation** = naming makes things governable (same instinct as seed.yaml contracts)
- **"or" = parallel path** not choice ‚Äî pre-loads all leaves so any branch activates instantly
- **PARAMETER = patch modulation** ‚Äî same design, different parameters = different color

## Relationship to Session Protocol

Session grammar (SEED‚ÜíHANDOFF) governs HOW you talk to AI (minutes‚Üíhours).
Design grammar (DEFINE‚ÜíTRANSFORM) governs HOW you make AI produce artifacts (hours‚Üídays).
One is the conductor's baton. The other is the composer's notation. They nest: a session CONTAINS a design sequence.

**Why:** Formalizing this means any agent can recognize which design phase the user is in and respond with the appropriate depth/format.

**How to apply:** When the user issues an imperative on a specific object (D1), they're starting a design sequence. When they introduce $VARIABLES (D3), the output crosses from conceptual to computable. When they set $DEPTH/$MODE (D7), they're parameterizing ‚Äî don't pick one, prepare both. When they say "tighten into" or "translate into" (D9), they want format transformation, not new content.
