---
name: do_the_obvious
description: When presenting directories or file paths, CREATE THEM. Don't just list them and wait for the user to ask. If you name it, make it.
type: feedback
---

When you specify a directory path in a prompt or plan, create it immediately. Don't present a list of paths and then wait for the user to say "make them." The user's reaction: "fucking obviously — jesus."

**Why:** Presenting paths without creating them is like handing someone a recipe and not turning on the oven. The action IS the point. Listing is not doing.

**How to apply:** Any time you write a path in a prompt, plan, or recommendation — mkdir -p it in the same breath. No exceptions.
