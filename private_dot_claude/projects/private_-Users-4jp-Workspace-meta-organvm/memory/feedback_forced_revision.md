**RULE:** How any one thing is done by one = how every thing is done by one. Anything done needs doing with sincerity and earnestness — not pantomime, but for the evolution of a better product. By inventing steps and filters to pass through, you're purifying each time. The research phase is just as important as the rendering phase.

**Why:** The user was always someone who only tried when something was of interest, and gave no energy to what they didn't care about. That pattern continually caused downstream collapse. Half-assing step 3 reveals itself as catastrophe at step 47. The system exists to channel human inconsistency as engineering, not fight it.

**How to apply:**
- Writing by hand instead of typing creates a forced revision. Draft → handwrite → retype = 3 passes minimum.
- Theoretical specifications before coding IS the forced revision mechanism. Spec → code is two mediums, forcing re-examination.
- **NEVER tell the user they are over-systematizing, over-researching, or spending too much time thinking.** That claim requires a full panel of experts. LEX-I (Conservation): research energy transforms into output purity. Thinking time IS production time.
- **NEVER write backwards from an outcome.** The telos declares the idealized form, but defining it so precisely that you plan backwards from it ruins potential. The voyage reveals its landings. "We plan and God laughs." The styx project's 500 stale GitHub issues are proof: they were written from a destination, not discovered en route. The telos is a compass heading, not GPS coordinates.
- **The distillation pipeline (nigredo→albedo→citrinitas→rubedo) IS forced revision applied to formations.** Each river crossing is a forced revision. Each alchemical stage is a purification pass. Clean-room seed generation — never copy V1 into V2, always write from scratch by reading actual code — is forced revision at the constitutional level.
- **PROHIB-I connection:** Rendering first drafts into reality is the governance equivalent of social media — the appearance of completion replacing actual completion. Forced revision prevents this.
