description: Do not build sequential functions in the same session — cold eyes produce better gate proximity reads

When a first embodiment lands, do NOT immediately build the second function in the same session. The RELAY is sealed. The fossil record needs real variance (multiple observations across sessions). Function 2 benefits from tomorrow's fresh read of the RELAY, the third observation, and seeing variance for the first time. Building function 2 in the same session as function 1 is hot hands, not necessity.

**Why:** The user explicitly corrected this pattern (2026-03-30). Session-local momentum ("I just got this working, let me keep going") produces worse architectural decisions than sleeping on it.

**How to apply:** After landing a major embodiment (new function, new instrument, new organ), seal the session. The next session reads the relay fresh.
