Patch-Based Signal Architecture — SOPs become living receivers, not static documents. Modular synthesis applied to process governance.

**Core:** Every artifact output emits a signal. SOPs declare parameter interfaces (physics/bio/chem/met constraint categories). Patches wire signals to SOPs with specific parameter values. Same SOP, different patches = different governance profiles.

**Why:** The system has SOPs (60+ documents), operational pattern taxonomy (15 patterns), and dispatch infrastructure — but no bridge connecting "artifact produced" to "SOP governs it." Every pipe output needs a replicable process preset.

**How to apply:** New `patches/` engine module. Patch registry (YAML data, not code). Signal graph queryable via CLI. Dashboard patch bay view. Revenue layer: patch packs are exportable products, custom patch design is consulting.
