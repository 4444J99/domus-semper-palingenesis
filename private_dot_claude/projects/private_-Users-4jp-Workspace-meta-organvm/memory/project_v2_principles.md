**Established S42 (2026-03-30).** These principles govern all V2 work.

**Prime Directive:** Every part must locate itself, sense its relations, inherit its memory, express its state, and carry a compressed image of the whole. Create a unified ontology, map all relations, preserve multi-scale legibility, maintain reflective world models, convert structure into sensation, distribute compressed wholeness across all nodes, synchronize the whole rhythmically through active memory. Three theoretical pillars: [1] ontology (everything belongs to one reality), [2] cybernetics (everything senses and regulates), [3] phenomenology (the system renders its own density into experience). Two instruments: [a] AMMOI, [b] ontologia + variables + temporal metrics.

**Minimalism = Signal Integrity:** Every directory is a wall. Every wall blocks signal. A wall exists only if proven necessary by executable code — not rhetoric, not convention, not feeling. The fewer containers, the more the system recognizes pattern across context. Classification lives in data (inside files), not in naming. Sub-types inside the file, not in the filename. One pattern applied everywhere with contextual tweaks.

**Modular Synthesis Architecture:** Functions are modules in a rack. Each does ONE verb. Patch cables (signal graph) connect them. Products EMERGE from patching — they carry assets (HTML, images, data) but borrow logic from function modules. A product is a saved patch, not a self-contained application. The rack IS the container (a-organvm/). The medium IS the communication. No walls between modules.

**Multiverse Principle:** `a-` is the indefinite article. `a-organvm` = an organvm, one instance among infinite potential. The architecture is the species definition. The instance is the individual. Anyone running the same genome (constitutional law) through the same river (distillation engine) gets a valid organvm — different formations, same structure. The `a-` is a variable.

**11 Biological Systems → ORGANVM:** lifecycle (preserve/transform/observe/honor-death), nervous (enforce/audit/promote), circulatory (route/map/bond), digestive (measure/index/atomize), endocrine (regulate/affect), immune (watch/verify/filter), skeletal (define/identify/record), muscular (execute/sync/build), respiratory (ingest), integumentary (present/expose), reproductive (distill/generate), memory (remember/witness/narrate). In V1: 9 of 11 were LIVE, 2 PARTIAL (excretory + reproductive — exactly the two V2 needs). Naming convention: `{system}--{function}`.

**Every Proof Must Compile:** No assertion without executable test. Directory existence defended by pytest, not persuasion. If the proof doesn't exist, the directory doesn't exist. Proofs by contradiction: test both hypotheses, measure what breaks. If both options work equally, the structure is arbitrary (feeling). If one breaks, the other is necessary (physics). The universe's language is numbers.

**8 Organs → 8 Capability Dimensions:** V1 departments → V2 dimensions. Theoria, poiesis, ergon, taxis, logos, koinonia, kerygma, self-knowledge are continuous participation values (0.0-1.0) in formation.yaml, COMPUTED from code analysis by digestive--atomize, not declared by a human. A formation participates in multiple capabilities simultaneously. No ownership. No containment. Participation.

**schema.org as Computed View:** formation.yaml is YAML with ORGANVM ontology. schema.org JSON-LD is generated by the API (dashboard/portal) on demand. The organism speaks schema.org through its skin (API), not in its cells (files). Never store schema.org as a separate file — that violates minimalism (two files for one identity).

**SemVer for the Organism:** genome versioning in generationes.yaml. V1 = 0.x.y (pre-release). V2 = 1.0.0 (first version that can reproduce). Adding a capability = MINOR. Removing/restructuring = MAJOR. Healing = PATCH. Each formation carries contract_version (schema SemVer) and formation_version (own state SemVer).

**Three-Dimensional Identity:** {ontological name} × {topological location} × {SemVer temporal state}. Filename carries WHAT. Path carries WHERE. Internal metadata carries WHEN. No generic names. Every name answers "what IS this?"
