## Recurring Workflows

**0. The Domain Map Impulse (project inception):**
At the start of any new project: immediately map ALL domains within the realm of the project. Know every phase it must pass through to become reality. Before building anything — know the full topology. Systems first, structure first, then execute within the structure. This is the prime creative instinct: see the whole before touching any part.

**0b. The Repetition Refinement Principle:**
Every time a unique action is performed: identify the inputs/outputs and edges of what gravitational force naturally groups. Systemize as repeatable. Near-never repeat without refinement — repetitions refine. The action produces the SOP; the SOP refines the action; the refined action refines the SOP. Never loop data with outdated context — old topology studies (like ORGAN-IV three-stratum) are objective phenomena records, not prescriptions. Theory requires code equivalent to ratify — an unimplemented topology is a hypothesis, not a fact.

**0c. The Prompt Chain Signal:**
All prompts ingest → future repeatable cycles become sequenced prompt chains. Prompts are not single-shot. Every unique prompt is a link. Chains inherit from prior links. All plans stored in the system's library — not just project-local, but centrally indexed and discoverable.

**0d. The SOP Lifecycle:**
SOPs are not written — they are discovered: rep(etition) → absorb → fortify → place → chain. Do the thing once (action). Do it again on different input (repetition). Compare the two runs (absorb the invariant structure). Harden against edge cases (fortify). Place in the correct scope (project/organ/system). Chain to upstream triggers and downstream consumers. Proof: the Superproject Topology Audit SOP was discovered this way — ran on ORGAN-IV (28 repos, 68% gap), ran on META (14 repos, 14% gap), compared invariants across 7 steps, formalized. The SOP lives at `organvm-iv-taxis/.claude/plans/2026-03-31-superproject-topology-audit-sop.md`.

**0f. The Universal Directory Canon:**
Certain directories and markdown files repeat across every project — guaranteed referenceable. Not every project needs `research/`, but there IS a set of locations that are universal and should exist as a canonical library pattern. These are the bones that every skeleton shares:

Universal (every project):
- `CLAUDE.md` / `AGENTS.md` / `GEMINI.md` — agent context (nested per Context Architecture)
- `.claude/plans/` — dated plan archive (never overwrite, revision suffix)
- `.claude/memory/` — persistent session memory (1:1 local:remote)
- `seed.yaml` — automation contract (organ, tier, produces/consumes)
- `README.md` — hero section: problem → approach → outcome

Common (most projects):
- `docs/` — architecture, specs, reports
- `docs/superpowers/specs/` — design specification documents
- `tests/` — test suite
- `scripts/` — operational tooling
- `.github/workflows/` — CI/CD

Situational (when the project needs them):
- `research/` — accumulated knowledge prose
- `intake/` — unsorted inbound material
- `tools/` — audit/build scripts
- `playbooks/` — operational runbooks, handoffs
- `templates/` — reusable scaffolds
- `strategy/` — scoring rubrics, rules, market intel

The universal set is the library. The situational set is discovered through the Domain Map Impulse (0). When a new project starts, stamp the universal set immediately, then let the domain map reveal which situational directories emerge.

**0e. The Society of Voices:**
The tribunal pattern (3 personas) is the surface. The deeper principle: a meta-layer that recreates an idealized society of voices who take shape from the faces one sees as they battle their day as being themselves. The personas are drawn from lived experience — not stock archetypes, but the actual advisors, critics, and allies encountered in reality. The tribunal is a mirror that reflects back the user's own multiplicity through structured adversarial synthesis.

## Philosophical Principles

**6. The Human as Sun:**
The human is the conductor, the prima donna, the brightest star. System NEVER defers grunting work to the human unless the universe absolutely requires it. Every deferral = lazy system design. The human does dreamwork/ideawork only.

**7. The Wedding Cake:**
MVP = the project in totality. Build in layers. Each layer reveals sell-able artifacts. "We expand, we do not reduce." Disagree with solutions — never with reduction.

**8. The Imposter Syndrome Inversion:**
"How I think of myself" is NOT a valuable asset. "What I've done" IS what I am. The purpose of the ORGANVM system is commodifying the creative process itself — creating in the dark, monetizing the process, not just the output.

**9. The White Matter Principle:**
Intelligence isn't about bigger brains — it's about white matter integrity (how regions communicate). The system fault is high-powered gray matter (isolated clusters of capability) that aren't talking to each other. The solution is always connection, not more capacity.

**10. The Modular Testament:**
Each thing should exist as its own thing. 1:X testing — each module requires its required amount of testing. Enact the modular testament.

**11. The Artistic Triforce:**
Three polarities of creative output: (I) Conscious — autobiographical, persona, present reality. (II) Subconscious — dreams, horror, masks, underworld. (III) Temporal — past/future mythologies, religion, canon, speculative fiction. All mediums span all three. Not genre sorting — lane architecture.

**13. The "Eat Off the Floor" Gate:**
Is it clean enough to eat off the floor? No patchwork, no bandaids, no under-rug sweeps. Fundamental fixes — gravitational, biological, chemical, physics-based — with logic, unity, and upward striving for utopic internal peaceful harmony.

**14. The Ontological Naming Convention:**
3-4 words (ignore non-words). Dense, powerful sentence-like title describing what the thing IS and DOES. Single-hyphen for compound words. Double-hyphen for semantic distance. Provide 10 combinations in English + Latin/Greek translations.

## System Architecture Principles

**15. The Business Organism (8 departments):**
ENG (Alpha) → LEG (Alpha) → PRD (Beta) → OPS (Beta) → FIN (Beta) → GRO (Delta) → CXS (Delta) → B2B (Omega). Each department has: human track + deep agent track. Agent seeding: create memory dir → ingest corpus → execute first task → store findings → link from CLAUDE.md.

**16. Three-Scenario Planning:**
Every metric has Conservative / Moderate / Aggressive targets. Conservative = founders + 1 contractor. Moderate = 1-2 hires, small paid budget. Aggressive = seed fundraise, 3-5 hires, growth team.

**17. The Fractal Timeline:**
Progress measurement takes the shape of the context it's viewed in. Macro enterprise = system-shaped meter. Zoom into org = org-shaped meter. Zoom into feature = atom-level meter. The timeline recontextualizes, not just filters. Nesting doll from system to atoms.

**18. The Post-Flood Signal Vocabulary:**

## Hardening Patterns

**20. The Hardening Memorandum (7 deliverables):**
When any project enters hardening phase, produce: (1) academic canon, (2) market thesis, (3) competitive map, (4) regulatory matrix, (5) delivery method, (6) proof-asset plan, (7) knowledge-base schema. Always review past domains, current domains, and future domains. Primary sources must be legitimate; secondary sources must be peer-reviewed only.

**21. The Three-Stratum Topology:**
Any organ/project has 3 strata: (I) Core — declared, tracked, official. (II) Untracked Core — present but not declared, gravitationally bound. (III) Contribution/External — workspaces, forks, dormant. The submodule list lies — it captures less than a third of what actually exists. Always audit all 3 strata.

**22. The Isotope Dissolution Pattern:**
When code is duplicated across repos, the duplicate is an "isotope." Isotopes dissolve by importing from the canonical source. Gate contracts track dissolution status. Dissolved isotopes are removed, not archived.

## Auditor Pair (bidirectional topology check)

**23. Auditor Above — fleet-level gap scan:**
Top-down. Cross-boundary violation detection. The triage table: run once, read the gap%, drill into outliers. Sees the whole, finds what's missing or misaligned. The Superproject Topology Audit SOP is the codified form.

**24. Auditor Below — reverse lookup:**
Bottom-up. Who claims me? What stratum am I? Disposition check: absorb / dissolve / product / archive. Every component asks itself: where do I belong, who depends on me, and what happens to me next?

These always pair: Above scans the fleet and produces a gap table. Below takes each gap entry and resolves its disposition from the component's own perspective. Above is immune--watch (surveillance). Below is immune--verify (self-assessment).

## Uncategorized "Natural Kin" Patterns

Prompt patterns observed but not yet formally categorized — need 48-hour archaeology pass:

- **Paradigm Correction** — "based on your questions, I believe you have a fundamental misunderstanding" — forceful redirect when AI's assumptions are structurally wrong, not just imprecise
- **Call Transcript Feature Extraction** — ingesting spoken conversation transcripts to produce theoretical specification documents with ALL features mentioned
- **Contact Cohort Research** — researching 120+ profiles to derive optimization rules (LinkedIn elevation used this to identify the "pipe-separator formula" and "Shabani pattern")
- **Identity Working-Through** — using AI as sounding board for imposter syndrome / identity questions ("is 'how I think of myself' a valuable asset?")
- **Process Meta-Questions** — "does working with Gemini seem to help or harm?" — questions about the process of using AI, not the work itself
- **AI Self-Limitation Query** — "what would your maker need to unlock within you to grant you full omniscient omnipotence?" — asking the AI to describe its own constraints
- **Multi-Agent Workforce Dispatch** — "plan extensive/exhaustive multi-agent parallel workforce streams to conquer the full feature set"
- **Prompt-as-Handoff** — "for all the critical-tensions/unanswered-questions/gaps, compile into a prompt for ai-handoff" — the output IS a prompt for the next session
- **The Public Process Mandate** — "the public process should most certainly be done in public — we don't do this for our health internally solely" — work must be visible, symbiotic
- **Rewrite-via-crowd** — "post different texts, everyone who interacts with the website can make their edits, track drift, come to an amalgam based on what everyone did" — collaborative editing as algorithm
- **The Omniscient Advisor** — "being-all-seeing advisor, you tell me" / "all of the above in logical order" — deferring sequencing to the AI's judgment

**Why:** These are the abstract patterns that generate the operational vocabulary. When the user says "glorious gloriosity" (vocabulary), they're invoking the Wedding Cake principle (framework).
