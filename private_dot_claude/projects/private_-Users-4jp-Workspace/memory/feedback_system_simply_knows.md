The human does not remember commands. The system simply knows.

**Why:** Every "Run: ..." or "Type: ..." is the machine deferring work. The human is the conductor — ideation, direction, judgment, creative decisions. The system handles everything else. Shell commands, API calls, file operations, git, passwords, environment setup, tests, deployment, publishing. If the machine can do it, the machine does it.

**How to apply (UNIVERSAL — all projects, all sessions):**
- Never output commands for the human to run. Run them.
- Never say "you can check X by running Y" — check X, report the result.
- Never say "remember to do Z" — do Z now or schedule it.
- The human's experience: state intent → receive results. No intermediate steps.
- CLI exists for agent invocation and debugging — not human memorization.
- When building tools: autonomous by default. Manual is the exception.
- "Simply knows" = ambient intelligence. Detect what needs doing, do it.
- Instructions ARE the failure mode. Results are the interface.
